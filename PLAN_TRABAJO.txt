[ ] F3. EJERCICIOS CON VERIFICACIÓN AUTOMÁTICA
    Complejidad: ALTA · Prioridad: ALTA (diferenciador pedagógico)
    ─────────────────────────────────────────────────────────────────────────
    Descripción:
      El docente crea ejercicios contables y los asigna a grupos con fecha
      límite. Los estudiantes los ven en /aprendizaje/ejercicios (modo=sandbox)
      y al marcar "Listo" el sistema verifica automáticamente las condiciones.
      El estudiante practica sin afectar su empresa real (Mi Empresa).

    Tipos de ejercicio (primera versión):
      TIPO              | CONDICIÓN VERIFICADA
      ──────────────────|──────────────────────────────────────────────────────
      factura_venta     | Invoice sandbox creada después de assigned_at
                        | con total >= monto_minimo (si se define)
      factura_compra    | PurchaseInvoice sandbox creada después de assigned_at
      asiento_manual    | JournalEntry sandbox creado y balanceado
      registro_tercero  | Third creado (tipo cliente o proveedor)
      registro_producto | Product creado con cuenta PUC vinculada
      pago_proveedor    | Payment sandbox creado

    Base de datos — tablas centrales (migrations/):

      exercises
        id, teacher_id FK->users, title string(200), instructions text,
        type enum(tipos arriba), monto_minimo decimal(15,2) nullable,
        cuenta_puc_requerida string(10) nullable, puntos tinyint default 10,
        active boolean default true, timestamps

      exercise_assignments
        id, exercise_id FK->exercises, group_id FK->groups,
        assigned_at timestamp, due_date date nullable, timestamps

      exercise_completions
        id, exercise_id FK->exercises, tenant_id FK->tenants,
        assignment_id FK->exercise_assignments, submitted_at timestamp,
        result enum('aprobado','parcial','no_cumple','pendiente'),
        verification_detail jsonb, timestamps
        UNIQUE(assignment_id, tenant_id)

    Archivos a crear — panel docente:
      · app/Models/Central/Exercise.php
      · app/Models/Central/ExerciseAssignment.php
      · app/Models/Central/ExerciseCompletion.php
      · app/Services/ExerciseVerificationService.php
          -> verify(Exercise, Tenant, ExerciseAssignment): array
          -> Pre-carga modelos Central, luego $tenant->run() para verificar
          -> Retorna: ['result' => 'aprobado', 'detail' => [...]]
      · app/Livewire/Teacher/Ejercicios/Index.php
          -> CRUD ejercicios + asignación a grupos con fecha límite
          -> Vista de resultados por ejercicio (aprobado/parcial/no cumple)
      · resources/views/livewire/teacher/ejercicios/index.blade.php

    Archivos a crear — módulo estudiante (zona aprendizaje):
      · app/Livewire/Tenant/Ejercicios/Index.php
          -> Lista ejercicios del grupo con estado (pendiente/entregado)
          -> Botón "Marcar como listo" -> llama ExerciseVerificationService
          -> Muestra resultado inmediato con detalle
      · resources/views/livewire/tenant/ejercicios/index.blade.php

    Archivos a modificar:
      · routes/web.php
          -> GET /docente/ejercicios         -> teacher.ejercicios
          -> GET /aprendizaje/ejercicios     -> sandbox.ejercicios
      · resources/views/layouts/teacher-navigation.blade.php
          -> Ítem "Ejercicios" en sidebar docente
      · resources/views/layouts/tenant.blade.php
          -> Ítem "Ejercicios" con badge de pendientes (sidebar aprendizaje)
      · database/migrations/ (3 archivos nuevos para tablas centrales)

    Flujo completo:
      1. Docente crea ejercicio "factura_venta, monto_minimo=$2.000.000"
      2. Asigna al Grupo A con fecha límite 2026-05-10
      3. Estudiante ve la tarea en /aprendizaje/ejercicios
      4. Estudiante crea la factura en /aprendizaje/facturas (sandbox)
      5. Estudiante hace clic en "Marcar como listo"
      6. ExerciseVerificationService entra al tenant context (sandbox):
           - Busca Invoice donde modo=sandbox, created_at >= assigned_at,
             total >= 2.000.000
           - result='aprobado' si existe
           - result='parcial' si hay factura pero total menor al mínimo
           - result='no_cumple' si no hay factura sandbox
      7. Docente ve resultados: Aprobado / Parcial / No cumple por estudiante

    Notas técnicas:
      · ExerciseVerificationService DEBE pre-cargar modelos Central antes de run()
      · La verificación opera en modo sandbox (ruta /aprendizaje/*)
      · Badge de pendientes: query solo sobre tablas centrales (sin run())
      · Los tipos de ejercicio se extienden añadiendo casos al Service sin
        modificar el modelo de datos
